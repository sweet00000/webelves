Forge — Knowledge synthesis · Ultra

A mini AI trainer you can run.

Give Forge a list of topics. It writes, grades its own work, discards the weak results, refines the rest, and packages everything into portable .elf files anyone can open in a browser — offline, forever. No GPU farm needed.

The four-stage loop
[1] Generate
Write a draft
For each topic, Forge writes a structured knowledge article using whatever model you have — cloud API or fully local.
[2] Judge
Grade it honestly
A discriminator reads the draft and scores it against your quality rubric. Pass or fail, with notes. No vibes — structured grades.
[3] Refine
Fix what's weak
Failed drafts get the critique as new instructions. Loop repeats until the article passes or hits the retry limit.
[4] Package
Build the .elf file
Accepted articles bundle into a portable .elf — self-contained, shareable, air-gap safe. Open in any browser, forever.
What you end up with
[A]
Portable .elf files
Each topic becomes a self-contained knowledge file. Share over USB, email, or LAN — recipient gets a full offline AI chat experience.
[B]
Training corpus
Accepted articles accumulate into a structured dataset — raw material for fine-tuning a small model that generates .elf files without any cloud API.
[C]
In-browser AI emitter
End goal: a fine-tuned 1–3B model that takes a topic and produces a new .elf entirely in a browser tab.
Open by design

The .elf format spec is public. Forge is pure Python — no Node, no cloud dependencies baked in. Run fully offline with a local model. Quality rubrics are configurable YAML files — you set the bar for your domain.

Medical literature, legal documents, private knowledge base — the pieces are all there. Not a hosted service with vendor lock-in. An open pipeline you can run, fork, and extend.

The format

What's in a .elf

A single portable file — HTML in v0.2, ZIP in v1.0 — bundling everything needed to open, browse, and chat with a knowledge base, entirely offline. No server. No install. Open it in any modern browser.

[01]
Manifest
RFC 8785 JCS-canonical JSON. SHA-256 over the manifest = the artifact's identity fingerprint. Same fingerprint forever — auditable without a server.
[02]
Corpus
Articles + passage chunks. Embeddings precomputed and bundled inline. Every answer cites a specific passage in a specific source you provided.
[03]
Runtime
A wllama (GGUF/WASM) or transformers.js (ONNX) bootstrap inlined in the file. Opens a full chat UI on load. No CDN dependency for core functionality.
[04]
Optional model
A small LLM (200MB–1GB) baked in for full offline use, or lazy-fetched for slim "lite" artifacts. The file works without a model — BM25 keyword search still runs.
NotebookELF

NotebookLM-style. Builds .elf.

A browser studio for producing portable knowledge files. Dashboard of notebook cards — each holds sources, grounded RAG chat with inline citations, and a notes studio. Hit Build and it packages everything into one self-contained .elf file.

[01]
Three-panel workspace
Sources panel · Chat panel · Studio panel. Ask anything, teacher cites every answer. Notes are editable and exportable.
[02]
Structured citations, always
Every answer carries machine-readable citation spans — traceable to your exact source passage. Not regex over freeform prose.
[03]
PDF · paste · URL
All processed in-browser via pdf.js and Readability. Content-addressed blobs in OPFS — nothing ever uploaded to a server.

How storage works

Your browser is the database.

No backend. No database server. No cloud bucket. Your notebooks, files, and artifacts live in browser storage APIs. We cannot access your data even if we wanted to.

[OPFS]
Origin Private File System
Notebooks, blobs, and built .elf outputs. Real filesystem API, scoped to this origin, sandboxed. The browser manages it — not us.
[IDB]
IndexedDB
Structured metadata, workspace state, embedding indexes. Survives restarts. Zero servers involved — Chrome or Firefox IS the database engine.
[LS]
localStorage
Settings, optional API keys, waitlist interest. Never transmitted. Clear any time from browser devtools — we have no copy.
[ZIP]
Export anytime
Everything in OPFS/IDB exports as a ZIP and re-imports on any machine. Portable because it never belonged to our servers.
On clearing browser data:  "Clear site data" deletes your notebooks — they live in your browser, not our servers. Export a ZIP backup first.
Hard commitments

What we won't compromise on.

  • [01]   No data leaves your machine. Ever. Model runs in your browser via WASM. Documents, queries, and outputs stay local. No cloud API receives them.
  • [02]   Browser-only at the consumer edge. Open a .elf in any modern browser. No install, no telemetry. Disconnect — it still works.
  • [03]   Offline-first. External calls gated by an explicit API key you provide. No key = no cloud call.
  • [04]   Manifest is identity. SHA-256 of the canonical manifest = the artifact's fingerprint. Same file, same fingerprint, forever. Auditable without a server.
  • [05]   Citations are structured. Machine-readable citation spans — traceable to your original source documents.
  • [06]   Forge is Python only. No Node deps. Browser tools stay browser-native.
